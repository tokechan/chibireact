# A.2 本家 React のソースコードを読む

本書を読み終えたあなたは、**Fiber / hooks / scheduler / Suspense / Concurrent** の概念を完全に理解しています。
ここから自然な次のステップは **本家 React のソースを読むこと** です。

この章は地図です。React リポジトリのどこに何があるか、どう読めば効率的かを案内します。
**本書の各章 → React の対応ファイル** の対応表を用意しました。

## この章のゴール

- React リポジトリの構造を把握する
- 自分の関心 (Hooks / Fiber / Suspense...) ごとにエントリポイントが分かる
- ソースを読むコツ (検索ワード / 用語 / コメント文化) を理解する
- 困ったら参照すべき外部リソースを知る

> 章末に動くもの: なし（読み方ガイド）。

前章: [A.1 30 分で React を作る (1 ファイル版)](./010-thirty-min-react)

## React リポジトリの全体構造

[github.com/facebook/react](https://github.com/facebook/react) のディレクトリ:

```
react/
├── packages/
│   ├── react/                  ← createElement, hooks API surface
│   ├── react-dom/              ← Web ターゲットの renderer (DOM 操作)
│   ├── react-reconciler/       ← Fiber / work loop / reconciliation の本体
│   ├── scheduler/              ← lane / 優先度 / 中断・再開
│   ├── react-server/           ← Server Components 関連
│   ├── react-server-dom-webpack/  ← RSC の Webpack 統合
│   ├── shared/                 ← 各パッケージ共通ユーティリティ
│   ├── react-devtools/         ← DevTools 拡張
│   └── ... (他に多数)
├── scripts/
└── fixtures/
```

**最も重要なのは** `react-reconciler/` と `react-dom/`。本書で言う Part 2-3 の核心がここにあります。

| 探したいもの | 検索ワード |

## 読むときの心得

### 1. 「全部理解しよう」と思わない

React のコードは **30 年分の試行錯誤** の塊です。一度に全部理解するのは無理。
本書を読んだあなたなら、**気になった機能 1 つだけを追う** で十分。

### 2. コメントが豊富 (機能フラグ周辺は特に)

```js
// We don't fall through to the regular workInProgress component.
// Suspense never matches.
```

英語のコメントは **設計判断の理由** を含むことが多い。翻訳しながら読むと学びが深い。

### 3. テストコードを併読する

`packages/react-reconciler/src/__tests__/` の各テストは **「この機能は実際にどう使うか」** の最高の例:

```bash
# 全テスト見るだけでも勉強になる
ls packages/react-reconciler/src/__tests__/
```

### 4. dev build と prod build の差を意識

```js
if (__DEV__) {
  // 開発ビルドだけのコード (warning, type check)
}
```

`__DEV__` ガード内のコードは production では消されます。**「警告系は読まなくていい」** と判断するのに使えます。

## ちょっと深掘り

### React 内部の用語集

本家コードを読むと出てくる固有名詞:

- **Reconciler**: React の差分計算エンジン (Fiber based)
- **Renderer**: 結果を反映するターゲット (react-dom / react-native / etc.)
- **Host**: renderer ターゲット (DOM の場合は HTMLElement)
- **Lane**: 31 ビット優先度
- **Effect**: commit phase で実行する副作用 (DOM 操作含む)
- **Hook**: function component に紐付く状態 (本書 Part 3)
- **Throw**: Promise / Error を発生させること (Suspense / ErrorBoundary)
- **Hydration**: SSR で出力した HTML に React を「上書き」する処理
- **Dispatcher**: hook の現在の実装 (mount/update で切り替わる)

### 「読むのが辛い」と感じたら

巨大コードを読むのは **必ず辛い瞬間が来ます**。そのときは:

- **本書に戻る** — 概念を再確認
- **小さい機能から** — 全体を諦め、自分が興味ある hook 1 つだけ
- **Issue / PR を読む** — 機能追加時の議論はコードより理解しやすい
- **JSer.dev** — 図解で全体像を掴む

「本家のコードを読み切ること」が目的ではなく、**「自分のコードがなぜ動くかを深く理解する」** ことが目的。本書 + 本家の組み合わせで十分達成できます。

### React Compiler (Forget) のソース

[github.com/facebook/react/tree/main/compiler](https://github.com/facebook/react/tree/main/compiler) に新しい Compiler のコードがあります。
React 本体とは別の独立パッケージで、**RxJS の Babel プラグイン** として動きます。

これは React の「**未来形**」なので、本書のスコープ外ですが、興味ある方には新天地です。
