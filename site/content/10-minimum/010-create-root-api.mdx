## React 18 の `createRoot` とは

React 17 まで、エントリポイントは `ReactDOM.render(<App />, container)` でした。
React 18 から **新しい API として `createRoot`** が導入されています。

```js
// React 17 まで（旧 API）
ReactDOM.render(<App />, document.getElementById('app'))

// React 18 以降（新 API）
const root = ReactDOM.createRoot(document.getElementById('app'))
root.render(<App />)
```

なぜ変わったかというと、**Concurrent Rendering を有効にするため**です。`render` を毎回呼ぶ古い API では、root に紐づく状態（中断中の更新、優先度など）を引き継げませんでした。新 API では `root` という概念が一級市民になり、Concurrent 機能の前提条件になっています。

本書では新 API である `createRoot` をそのまま採用します。

import { describe, it, expect } from 'vitest'
import { createRoot } from '../src/create-root'

describe('createRoot', () => {
  it('container を渡すと render メソッドを持つ root オブジェクトを返します', () => {

## ちょっと深掘り

### 本家 React の `createRoot` を覗いてみる

[React の本家ソース](https://github.com/facebook/react/blob/main/packages/react-dom/src/client/ReactDOMRoot.js) では `createRoot` は数百行に渡る複雑な処理ですが、本質的には:

1. `container` から **FiberRoot** を作る
2. **HostRoot Fiber** を初期化
3. `ReactDOMRoot` クラスのインスタンスを返す（`render` / `unmount` メソッドを持つ）

本書では Fiber や FiberRoot をまだ知りません。それらは Part 2 で扱います。**今は形だけ真似る** ので OK です。

### `void container` ってなに？

TypeScript の strict モードで「未使用の引数」エラーを抑えるイディオムです。`container` を「使う気はあるけど今は使わない」と明示します。次の章で実際に使うようになると、`void` は外して通常の参照に置き換えます。
