## 1. JSX

JSX は **JavaScript 構文の拡張**で、HTML 風のタグを JS 式として書ける記法です。

```jsx
const element = <h1 className="title">Hello</h1>
```

これは Babel や SWC のようなコンパイラによって、以下のような `React.createElement` 呼び出しに**変換されます**:

```js
const element = React.createElement('h1', { className: 'title' }, 'Hello')
```

つまり JSX は構文上の砂糖（シンタックスシュガー）であり、**ランタイムには JSX という概念は存在しません**。本書 Part 4 でこの変換の仕組みを最小実装します。

## 2. 仮想 DOM (Virtual DOM)

仮想 DOM は、UI を表現する**軽量な JavaScript オブジェクト**です。実 DOM (HTML 要素) ではありません。

```js
// 実 DOM:  <h1 class="title">Hello</h1>
// 仮想 DOM: { type: 'h1', props: { className: 'title' }, children: ['Hello'] }
```

なぜ仮想 DOM が必要か:

- **実 DOM の操作は遅い**（リフロー・リペイントが発生します）
- **小さな変更を毎回反映するとパフォーマンスが落ちます**
- **JS オブジェクト同士の差分計算は速いです**

React は「次に作りたい UI」と「現在の UI」をどちらも仮想 DOM で持ち、**差分だけを実 DOM に反映**します。本書 Part 1 でこの仕組みを最小実装します。

## 3. Reconciler (Fiber)

Reconciler は仮想 DOM の差分計算を担うエンジンです。React 16 (2017) から **Fiber アーキテクチャ**として書き換えられました。

Fiber は次の特徴を持ちます:

- 描画作業を**小さな単位（Fiber Node）に分割します**
- 各単位の処理ごとに「ブラウザの応答性に余裕があるか」を確認します
- 重い更新を**中断・再開**できます
- ユーザー入力など緊急の更新を**割り込ませられます**

これが React 18 の **Concurrent Rendering** の基礎になっています。

| 旧 (Stack Reconciler, ~v15) | 新 (Fiber, v16+) |
|---|---|
| 同期的な再帰でツリーを辿る | 非同期で中断可能 |
| 一度始めると最後まで止まらない | 緊急の更新を優先できる |
| 巨大なツリーで UI が固まる | スムーズに保てる |

Fiber は本書 **Part 2 の主役**です。なぜ生まれたか、どう動くかを最小実装で追います。

## 4. Renderer (描画先)

Reconciler が「次にこう描くべき」と決めた変更を、**実際の出力先に反映する**部分です。React 本体は描画先を持たず、Renderer が分離されています。

| Renderer | 出力先 |
|---|---|
| `react-dom` | Web の DOM |
| `react-native` | iOS / Android のネイティブ UI |
| `react-three-fiber` | WebGL (3D) |
| `ink` | ターミナル |
| `react-pdf` | PDF |

この分離があるからこそ、React は多様なプラットフォームに展開できました。本書では `react-dom` 相当（Web の DOM）のみを扱います。

## 5. Hooks

Hooks は、**関数コンポーネントで状態や副作用を扱うための仕組み**です。React 16.8 (2019) で導入されました。

```jsx
function Counter() {
  const [count, setCount] = useState(0)
  useEffect(() => { document.title = `Count: ${count}` }, [count])
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```

主な Hooks:

| Hook | 用途 |
|---|---|
| `useState` | コンポーネントローカルな状態 |
| `useEffect` | コミット後の副作用（DOM 操作、購読） |
| `useContext` | Context API 経由の値取得 |
| `useReducer` | 複雑な状態遷移 |
| `useMemo` / `useCallback` | メモ化 |
| `useRef` | ミュータブルな参照 |
| `useTransition` / `useDeferredValue` | Concurrent 系 |
| `use` (v19) | Promise / Context の統合的な解決 |

Hooks は React の主役機能であり、**本書 Part 3 全体**を費やして実装します。
