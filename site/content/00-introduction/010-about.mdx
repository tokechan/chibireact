# はじめに

本書は「**React を小さく作りながら学ぶ本**」です。React の利用方法を解説する入門書ではなく、React の内部構造を**実装ベースで理解する**ための連載書籍です。

## なぜこの本を書いているのか

React は世界で最も使われている UI ライブラリの一つですが、内部実装を体系的に学べる日本語教材は驚くほど少ない状況です。

- 入門書は「使い方」止まりで、`useState` の中身までは踏み込みません
- 海外には [Build your own React (Pombo)](https://pomb.us/build-your-own-react/) という名著がありますが、2019 年公開で **`useState` のみ**しかカバーしておらず、Hooks 全体や Concurrent Rendering、Suspense は扱われていません
- [JSer.dev](https://jser.dev/series/react-source-code-walkthrough/) は最新ですが、本家ソースの**読解**スタイルで、写経しながら手で動かす本ではありません

一方、Vue 界隈には [chibivue](https://book.chibivue.land/) という素晴らしい教材があります。「**Vue を小さく作る**」という体験を通じて Vue の内部を理解できる、日本語の連載書籍です。

> 「**chibivue の React 版があったら**いいのに。日本語で、写経しながら、React 18 以降の主要機能を一冊で学べるもの。」

そう思ったことが本書を書く動機です。

## 想定読者

| 想定 | 具体的なイメージ |
|---|---|
| ✅ React を実務で使っている中級者 | コンポーネント設計や hooks は書けるが、内部の動きは知らない |
| ✅ Vue で chibivue を読んだ人 | 同じ思想の React 版を求めている |
| ✅ 「フロントエンドの仕組みが気持ち悪い」人 | ブラックボックスを開けたい欲求がある |
| ❌ React 完全初心者 | まずは [公式チュートリアル](https://react.dev/learn) を |
| ❌ React コミッター志望のエキスパート | 本書は「最小実装」止まり、本家ソースの全体把握には [JSer.dev](https://jser.dev/) を |

## 進め方

本書では各章で**動くコードを実際に書きます**。読むだけではなく、ぜひ手を動かしてください。

### 1. テストから書く（TDD）

各章の冒頭で「期待する振る舞い」を Vitest のテストとして書き、それを通すように実装します。これは [Pombo の "Build your own React"](https://pomb.us/build-your-own-react/) との明確な差別化軸です。テストがあることで:

- 章ごとに「動いた」が客観的にわかります
- 実装を変更しても壊れていないか即確認できます
- 「振る舞い」と「実装」を分けて理解できます

### 2. 章末でコミットして次へ

各章の終わりに `git commit` を切る運用を推奨します。後から特定の章の状態に戻りたくなった時、あるいは複数の章にわたる変更を比較したい時に役立ちます。

### 3. わからなくなったら本家を見る

迷ったら React 公式の [GitHub リポジトリ](https://github.com/facebook/react) を覗いてみてください。本書では「最小実装」を目指していますが、本家は本番品質のコードです。両者を比較すると「なぜ本家がこう書いているのか」が見えてきます。

## 環境構築のサマリ

詳細は [本書の進め方と環境構築](./040-setup-project) で扱いますが、必要なものを先に挙げておきます:

- **Node.js 22 以上**
- **pnpm 10 以上**
- **Git** + **GitHub アカウント**
- お好きなエディタ（VS Code / Cursor / IntelliJ など）

## 本書のスコープ外

正直に書いておきます。本書では以下は扱いません:

- React Native（Web 限定）
- Redux / Zustand などの状態管理ライブラリ（React 本体の話のみ）
- Tailwind / styled-components などのスタイリング
- Next.js / Remix などのフレームワーク（必要な範囲で SSR には触れる）
- パフォーマンスチューニングの実務テクニック（仕組みの理解までで止める）
